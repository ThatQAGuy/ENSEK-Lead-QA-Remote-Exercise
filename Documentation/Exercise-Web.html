Web Testing Exercise

We have provided you with a URL link to a test system which is to be used for this test: https://ensekautomationcandidatetest.azurewebsites.net/

Test Plan Generation Task

Based on the above application, you are required to create an approach for the testing that you are going to perform. This approach can be in a format of your choice, and it must clearly outline what you will be testing.

Test Plan Execution Task

Once you have completed the creation of your approach, we would then like you to execute one of your test cases against the application.

You will be required to record your results and test evidence for the test you execute, along with details of any defects that you encounter during your testing.

You can again choose to record the defects in any suitable format.

Assumptions

Below I have documented the assumptions I have made during the test planning and testing phase.

No additional applications are required to be run by the tester to provide functionality to the system under test.
A user must be logged in to successfully complete a buy order.
A user should not be able to buy more energy than is in stock.
A user should not be able to successfully complete a buy order for 0 units.
A user should not be able to successfully complete a buy order for negative units.
The application is built for desktop use and not mobile / tablets.
The sell energy page is under maintenance.
The contact page is a design choice and is not actually in an errored state.
